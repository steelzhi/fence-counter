Бэкенд приложения для расчета стоимости заборов

Алгоритм работы:
1. Пользователь выбирает на сайте товары и их количество.
2. Фронтенд передает на бэкенд данные о городе, который выбран для расчета (цены в городах отличаются), а также данные
по товару в формате json (id товаров и их количество).
3. Бэкенд обрабатывает полученные данные. При этом на стороне бэкенда производится обработка введенного пользователем
количества (по умолчанию, количество товаров, не кратное одной единице, округляется до целого в большую сторону).
4. По результатам обработки данных бэкенд формирует расчет, содержащий список товаров с их параметрами и ценами за
общее количество каждого товара, а также общую сумму заказа. Эти данные в формате json передаются на фронтенд.
5. Фронтенд отображает пользователю рассчитанные параметры его заказа.

Примечания:
- На хостинг сайта ежедневно выкладывается csv-файл со списком и параметрами товаров компании. При этом в одном столбце
такого файла может находиться сразу несколько параметров, что не подходит для работы с ним приложения Fence Counter.
К тому же в файл на хостинге выгружаются абсолютно все товары компании - многие из них не нужны для работы приложения.
Поэтому первым делом 1 раз в день (посредством установления задержки на главной нити, равной 1 дню) приложение парсит
файл на хостинге и создает файл в нужном для дальнейшей с ним работы формате и только с теми товарами, которые нужны
для расчета. Такой файл сохраняется на сервере компании, где также расположено само приложение. Фронтенд работает уже с
этим файлом.
- Бэкенд приложения размещается в Docker-контейнере на сервере компании.

Примененные технологии:
- Фреймворк Spring Boot;
- Docker;
- Postman.
