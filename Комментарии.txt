1. Сложность: в csv-файле параметры товаров, в основном, содержатся в названии этих товаров. Причем у разных товаров (профнастил, столбы, трубы профильные, заглушки) параметры будут разнотипными. Поэтому параметры товаров придется парсить из их названий.
Можно сделать так: приложение загружает себе файл .csv, выбирает из него доступные для расчета забора позиции, обрабатывает данные - парсит размеры позиций - и загружает новый .csv-файл на сервер. Фронтендер берет данные для отображения на сайте уже из такого, нового .csv-файла.
Примечание: приложение должно обновлять такой файл на сервере по расписанию (1/d).
2. Параметры выбранных клиентом товаров удобно передавать в бэкенд в виде RequestBody (json), а отдавать обратно в виде ResponseBody. При этом т.к. цена товара зависит от города, то нужно в параметрах пути передавать город.
